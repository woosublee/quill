# 재전사 복구와 범용 파일 저장 디자인

- 날짜: 2026-07-21
- 상태: 승인됨
- 범위: Note Browser의 전사 실패 복구, pill 버튼 상태, 범용 파일 저장, 기존 Obsidian 내보내기의 임시 이동

## 1. 배경

현재 다음 흐름에서 사용자가 녹음 유실을 걱정할 수 있다.

1. 로컬 전사 모델을 설치하고 전사 모델로 선택한다.
2. 선택된 로컬 모델을 삭제한다.
3. 녹음을 시작하고 종료한다.
4. 전사가 실패하며 Note Browser에 로컬 모델 누락 오류가 표시된다.

실제 구현은 실패 시에도 녹음 파일을 앱 저장소에 보존하고 `PipelineHistoryItem.audioFileName`으로 기록에 연결한다. Note Browser도 파일이 존재하면 오디오 플레이어를 표시한다. 그러나 현재 UI는 이 사실을 충분히 설명하지 않는다.

직접적인 원인은 오류의 대표 복구 동작과 재전사 가능 여부가 결합되어 있기 때문이다.

- `localModelMissing`의 `recoveryAction`은 `openModelsSettings`이다.
- Note Browser의 재전사 버튼은 `recoveryAction == retryTranscription`일 때만 나타난다.
- 따라서 녹음 파일이 보존되어 있어도 재전사 버튼이 사라진다.

같은 화면의 복사와 기존 공유 버튼도 전사문이 없으면 아무 동작이 없는 것처럼 보인다. 기존 공유 동작은 모든 사용자가 Obsidian과 Gemini를 사용한다는 가정으로 설계되어 일반적인 파일 저장 요구를 충족하지 못한다.

## 2. 목표

- 전사에 실패해도 녹음 파일이 보존되었음을 명확히 알린다.
- 사용자가 재전사에 필요한 조건을 이해할 수 있게 한다.
- 녹음 파일이 있으면 오류의 대표 복구 동작과 관계없이 재전사 진입점을 유지한다.
- 실행할 수 없는 동작이 아무 반응 없이 끝나지 않게 한다.
- 전사문이나 녹음 파일을 특정 앱에 종속되지 않은 방식으로 외부 폴더에 저장할 수 있게 한다.
- 기존 Obsidian·Gemini 사용자의 기능을 당장 제거하지 않는다.
- 기존 녹음 보존, 재전사, 삭제 흐름을 최대한 재사용한다.

## 3. 비목표

이번 범위에서는 다음을 하지 않는다.

- 로컬 모델 자동 설치
- 다른 전사 모델로의 묵시적 자동 전환
- 모델 설치 또는 선택 직후 자동 재전사
- Apple Live를 저장 파일 재전사 엔진으로 확장
- Obsidian 연결의 전면 재설계
- AI 요약 제공자와 UX의 전면 재설계
- 기존 녹음 저장 수명 주기의 변경

## 4. 선택한 제품 방향

검토한 방향은 다음과 같다.

1. 재전사 버튼만 복원하는 최소 수정
2. 복구 안내, 버튼 상태, 범용 파일 저장을 함께 개선하고 기존 통합은 임시로 보존
3. Obsidian과 AI 요약까지 한 번에 전면 재설계

두 번째 방향을 채택한다. 당장 사용자가 겪는 불안과 막힌 동작을 함께 해결하면서, 독립적인 통합 기능 재설계로 범위가 커지는 것을 막을 수 있기 때문이다.

## 5. 상태 모델

Note Browser는 오류의 대표 복구 동작과 노트가 현재 수행할 수 있는 기능을 분리한다.

### 5.1 독립적인 기능 상태

각 노트에서 다음 상태를 별도로 계산한다.

- `hasStoredAudio`: `audioFileName`이 있고 실제 파일도 존재한다.
- `hasTranscriptText`: 표시할 전사 텍스트가 공백 제거 후 비어 있지 않다.
- `retryAvailability`: 현재 선택된 전사 설정으로 저장 파일을 재전사할 수 있는지 나타낸다.
- `canExportAnyFile`: 전사 텍스트나 실제 녹음 파일 중 하나 이상이 존재한다.

`retryAvailability`는 최소한 다음 상태를 구분한다.

- `noAudio`: 재전사할 실제 녹음 파일이 없다.
- `unavailable`: 녹음은 있지만 현재 선택된 모델이나 자격증명으로 재전사할 수 없다.
- `ready`: 현재 선택된 로컬 또는 API 파일 전사 모델로 재전사할 수 있다.

기존 `QuillUserIssueRecord.recoveryAction`은 오류 카드의 대표 동작을 결정하는 역할만 유지한다. 재전사 버튼 표시 여부를 결정하는 데 사용하지 않는다.

### 5.2 재전사 지원 모델

저장된 파일의 재전사에는 다음 선택지만 사용한다.

- 설치되어 있고 준비된 로컬 Whisper 모델
- 필요한 API 자격증명이 준비된 Standard API 전사 모델
- 기존 파일 전사 경로가 지원하는 legacy local Whisper 모델

Apple Live는 실시간 녹음 전사 경로이므로 저장된 녹음의 재전사 모델로 간주하지 않는다.

현재 선택이 지원되지 않더라도 다른 모델로 자동 전환하지 않는다. 사용자가 Models 설정에서 로컬 또는 API 모델을 설치하거나 명시적으로 선택해야 한다.

## 6. 전사 실패 복구 UI

### 6.1 모델이 준비되지 않은 상태

녹음 파일이 실제로 존재하고 현재 재전사 모델이 준비되지 않았다면 Note Browser 오류 영역에 다음 내용을 표시한다.

**제목**

> 재전사 가능한 모델이 필요합니다

**보존 안내**

> 녹음 파일은 안전하게 보관되어 있습니다.

**해결 안내**

> 모델 설정에서 로컬 또는 API 전사 모델을 설치하거나 선택하세요. Apple Live는 저장된 녹음의 재전사를 지원하지 않습니다.

**대표 버튼**

> 모델 설정 열기

이 문구는 Note Browser의 저장 녹음 복구 맥락에만 적용한다. Settings나 Setup 등 다른 위치에서 쓰이는 일반 `localModelMissing` 문구를 녹음 파일 전용 문구로 바꾸지 않는다. Note Browser는 저장 자산과 현재 모델 상태를 함께 반영한 별도의 복구 presentation을 구성한다.

### 6.2 재전사 버튼

- 실제 녹음 파일이 있으면 pill에 재전사 버튼을 항상 표시한다.
- `retryAvailability == ready`이면 기존 `AppState.retryTranscription(item:)` 흐름을 실행한다.
- `retryAvailability == unavailable`이면 버튼은 클릭 가능하게 유지하고 전사는 시작하지 않는다.
- 이때 pill 위에 다음 토스트를 표시한다.

> 현재 재전사할 수 있는 모델이 없습니다.

화면의 지속 문구가 해결 방법을 설명하고, 토스트는 클릭이 실행되지 않은 이유만 짧게 설명한다. 두 문구는 같은 내용을 반복하지 않는다.

### 6.3 모델 준비 후 상태

사용자가 Models 설정에서 재전사 가능한 로컬 또는 API 모델을 준비하면 Note Browser의 기능 상태가 즉시 다시 계산되어야 한다.

저장된 오류 기록 자체는 재전사 성공 전까지 유지하지만, 화면에는 더 이상 모델이 없다는 오래된 안내를 표시하지 않는다. 대신 다음과 같이 현재 상태를 반영한다.

- 제목: `전사를 다시 시도할 수 있습니다`
- 본문: `녹음 파일은 안전하게 보관되어 있습니다.`
- 안내: `재전사 버튼을 눌러 다시 시도하세요.`
- 대표 동작: 재전사

모델 준비만으로 자동 재전사하지 않는다. 사용자가 동일한 노트에서 명시적으로 재전사를 실행한다.

### 6.4 오디오 파일이 실제로 없는 상태

기록에 `audioFileName`이 있더라도 실제 파일이 없다면 “안전하게 보관되어 있습니다”라고 표시하지 않는다.

- 오디오 플레이어를 표시하지 않는다.
- 재전사와 녹음 파일 저장을 비활성화한다.
- 기존 `audioUnreadable` 계열의 안전한 사용자 오류 문구를 표시한다.
- 존재하지 않는 파일을 복구하거나 삭제했다고 가장하지 않는다.

## 7. pill 버튼 동작

### 7.1 재전사

- 실제 녹음이 있으면 노출한다.
- 모델이 준비되지 않은 경우에도 클릭 가능한 상태로 두어 토스트 피드백을 제공한다.
- 재전사 진행 중에는 기존 진행 상태와 중복 실행 방지를 유지한다.

### 7.2 복사

- `hasTranscriptText == true`일 때만 활성화한다.
- 텍스트가 없으면 명확하게 흐린 상태로 비활성화한다.
- 도움말은 다음과 같다.

> 복사할 전사 텍스트가 없습니다.

복사는 대상 자체가 없을 때 실행할 수 없는 명령이므로 비활성 버튼 클릭 토스트는 제공하지 않는다.

### 7.3 파일로 저장

기존 공유 버튼의 기본 역할을 범용 `파일로 저장`으로 교체한다.

- 전사 텍스트나 실제 녹음 파일 중 하나 이상이 있으면 활성화한다.
- SF Symbol은 `square.and.arrow.down`을 사용한다.
- 도움말과 접근성 이름은 모두 `파일로 저장`으로 통일한다.
- 저장은 원본을 외부 폴더로 복사하는 동작이다.
- Quill 내부 녹음이나 전사 파일을 이동하거나 삭제하지 않는다.

### 7.4 더보기

pill에 더보기 메뉴를 제공하고 기존 Obsidian 내보내기를 이 메뉴로 이동한다.

- SF Symbol은 `ellipsis`를 사용한다.
- 메뉴 항목: `Obsidian으로 내보내기`
- 기존 Obsidian 내보내기와 그 안의 Gemini 요약 흐름은 당장은 보존한다.
- 현재 기능 조건과 오류 처리는 가능한 한 그대로 유지한다.
- 향후 Obsidian 연결과 AI 요약을 독립적인 기능으로 재설계한다.

삭제 버튼과 명시적 삭제 확인 흐름은 기존 동작을 유지한다.

### 7.5 토스트 공통 동작

- 토스트는 floating pill 바로 위 중앙에 표시한다.
- 기본 노출 시간은 2.5초로 한다.
- 새 토스트가 발생하면 이전 토스트를 즉시 교체하고 노출 시간을 다시 시작한다.
- 여러 토스트를 대기열로 쌓지 않는다.
- 토스트 내용은 VoiceOver가 읽을 수 있도록 접근성 알림으로도 전달한다.
- 창 크기가 좁아도 본문이 잘리지 않도록 최대 너비와 줄바꿈을 허용한다.

## 8. 범용 파일 저장 창

### 8.1 저장할 항목

Quill 전용 저장 창은 다음 항목을 제공한다.

- `전사 텍스트`
- `녹음 파일`

실제로 존재하는 항목은 기본적으로 선택한다. 존재하지 않는 항목은 비활성화하고 이유를 표시한다.

전사 텍스트가 없는 경우:

> 전사된 텍스트가 없습니다.

실제 녹음 파일이 없는 경우:

> 저장된 녹음 파일을 찾을 수 없습니다.

하나 이상의 항목이 선택되어야 저장할 수 있다.

### 8.2 전사 텍스트 형식

- 기본 형식: `일반 텍스트 (.txt)`
- 선택 형식: `Markdown (.md)`
- 새 저장 창을 열 때마다 `.txt`를 기본값으로 사용한다.
- `.txt`와 `.md` 모두 현재 전사 내용을 UTF-8로 저장한다.
- `.md`는 Markdown을 사용하는 도구가 해당 내용을 Markdown 문서로 취급할 수 있도록 확장자를 제공한다.
- 전사 텍스트가 없거나 선택되지 않았으면 형식 선택기를 비활성화한다.

### 8.3 파일 이름

- 현재 노트 제목을 기본 이름으로 사용한다.
- 사용자가 저장 전에 공통 기본 이름을 수정할 수 있다.
- 제목이 없으면 녹음 날짜와 시간을 사용한다.
- 경로 구분자, 제어 문자 등 파일명으로 사용할 수 없는 문자는 안전하게 정리한다.
- 정리 결과가 비어 있으면 날짜·시간 기반 이름으로 되돌린다.

두 항목을 선택한 경우 같은 기본 이름을 사용한다.

```text
2026-07-21 회의 녹음.txt
2026-07-21 회의 녹음.wav
```

녹음 파일은 원본 확장자와 바이트를 보존한다.

### 8.4 저장 위치

- 마지막으로 성공적으로 사용한 폴더를 기본 위치로 기억한다.
- 사용자는 `변경`을 통해 매번 다른 폴더를 선택할 수 있다.
- 마지막 폴더가 삭제되었거나 접근할 수 없으면 Documents 등 안전한 기본 위치로 전환하고 안내한다.
- 폴더 접근 권한을 지속해야 하는 앱 실행 환경에서는 기존 프로젝트의 파일 접근 방식에 맞춰 security-scoped bookmark를 사용한다.

표준 `NSSavePanel`은 한 번에 파일 하나만 저장하므로, 한 파일과 복수 파일에 동일한 경험을 제공하기 위해 Quill 저장 창에서 항목, 이름, 위치를 함께 관리한다. 폴더 변경에는 macOS 폴더 선택 패널을 사용한다.

### 8.5 저장 전 미리보기와 충돌

저장 창은 생성될 파일명을 미리 보여준다.

```text
2026-07-21 회의 녹음.txt
2026-07-21 회의 녹음.wav
```

같은 이름의 파일이 이미 존재하면 덮어쓰기 전에 충돌하는 파일을 알려주고 확인을 받는다. 사용자가 취소하면 기존 파일과 Quill 내부 원본을 변경하지 않는다.

### 8.6 저장 결과

성공 결과는 pill 위의 짧은 토스트로 표시한다.

- `2개 파일을 저장했습니다.`
- `녹음 파일을 저장했습니다.`
- `전사 텍스트를 저장했습니다.`

일부만 실패한 경우:

- 성공한 파일은 유지한다.
- 저장 창 안에 성공 항목과 실패 항목을 구분해 표시한다.
- 실패 원인을 안전한 사용자 문구로 설명한다.
- 선택과 파일명, 폴더 상태를 유지해 다시 시도할 수 있게 한다.

모두 실패한 경우 파일이 생성되지 않았음을 명확히 알리고 저장 창을 유지한다.

## 9. 데이터 흐름

### 9.1 재전사

1. Note Browser가 실제 오디오 파일 존재 여부와 현재 전사 설정을 평가한다.
2. 재전사 버튼 클릭 시 `retryAvailability`를 다시 확인한다.
3. 준비되지 않았다면 토스트만 표시한다.
4. 준비되었다면 기존 retry snapshot과 `AppState.retryTranscription(item:)` 경로를 사용한다.
5. 성공하면 기존 history item을 갱신하고 전사문을 표시한다.
6. 실패하면 안전한 사용자 오류를 기록하되 오디오 파일을 유지한다.

### 9.2 파일 저장

1. Note Browser가 현재 표시 전사문과 실제 오디오 URL로 저장 가능한 항목을 만든다.
2. 저장 창이 가능한 항목, 기본 이름, 기본 폴더를 표시한다.
3. 사용자가 항목, 텍스트 형식, 이름, 폴더를 확정한다.
4. 충돌 여부를 검사하고 필요하면 덮어쓰기 확인을 받는다.
5. 텍스트는 UTF-8로 쓰고 오디오는 원본을 복사한다.
6. 성공한 마지막 폴더를 기억한다.
7. 결과를 토스트 또는 저장 창의 상세 오류로 표시한다.

외부 저장은 history item과 내부 오디오 참조를 변경하지 않는다.

## 10. 오류 처리 원칙

- 재전사 실패는 녹음 삭제 조건이 아니다.
- 외부 저장 실패는 내부 원본 삭제 또는 이동 조건이 아니다.
- 사용자가 노트를 명시적으로 삭제할 때만 기존 삭제 흐름을 통해 연결된 내부 자산을 제거한다.
- 사용할 수 없는 동작은 명확히 비활성화하거나, 해결 가능한 선행 조건이 있는 경우 클릭 피드백을 제공한다.
- 화면의 지속 안내와 토스트 문구는 같은 정보를 반복하지 않는다.
- 저장 실패의 원시 경로나 내부 진단을 사용자 문구에 그대로 노출하지 않는다.

## 11. 테스트 전략

### 11.1 상태 행렬

다음 조합을 검증한다.

- 오디오 있음 / 텍스트 없음 / 모델 없음
- 오디오 있음 / 텍스트 없음 / 준비된 로컬 모델
- 오디오 있음 / 텍스트 없음 / 준비된 API 모델
- 오디오 있음 / Apple Live 선택
- 오디오와 텍스트 모두 있음
- 텍스트만 있음
- 기록에는 오디오 파일명이 있지만 실제 파일은 없음
- 오디오와 텍스트 모두 없음

### 11.2 재전사

- `localModelMissing` 오류여도 실제 오디오가 있으면 재전사 버튼이 표시된다.
- 모델이 준비되지 않은 상태에서 클릭하면 전사가 시작되지 않고 올바른 토스트가 표시된다.
- Apple Live 선택 상태에서 저장 파일 재전사가 시작되지 않는다.
- 준비된 로컬 또는 API 모델 선택 후 같은 노트를 재전사할 수 있다.
- 설정 변경만으로 자동 재전사가 시작되지 않는다.
- 성공 후 오류 presentation이 사라지고 전사문이 표시된다.
- 반복 실패 후에도 오디오 파일과 history 참조가 유지된다.

### 11.3 pill 버튼

- 텍스트가 없을 때 복사 버튼이 명확히 비활성화된다.
- 복사 비활성 도움말이 올바르게 지역화된다.
- 오디오만 있는 실패 노트에서도 `파일로 저장`이 활성화된다.
- 저장 가능한 자산이 없으면 `파일로 저장`이 비활성화된다.
- Obsidian 내보내기가 더보기 메뉴에서 계속 접근 가능하다.

### 11.4 파일 저장

- `.txt`, `.md`, 오디오 단독, 텍스트 단독, 복수 파일 저장을 검증한다.
- 공통 기본 이름과 확장자가 올바르게 생성된다.
- 위험한 파일명 문자가 안전하게 정리된다.
- 녹음 원본 확장자와 바이트가 보존된다.
- 마지막 성공 폴더가 다음 저장 창의 기본 위치로 사용된다.
- 접근할 수 없는 이전 폴더에서 안전한 기본 폴더로 복구된다.
- 중복 파일 덮어쓰기 확인과 취소가 기존 파일을 보존한다.
- 일부 저장 실패 시 성공한 파일과 실패한 파일이 정확히 보고된다.
- 외부 저장 후 Quill 내부 녹음과 history 참조가 그대로 유지된다.

### 11.5 GUI 확인

- 오류 카드의 문구 계층과 오디오 플레이어가 함께 보여 녹음 보존을 이해할 수 있는지 확인한다.
- 재전사 차단 토스트와 저장 성공 토스트가 pill 위에서 잘리지 않는지 확인한다.
- 복사 비활성 상태가 단순한 무반응 버튼으로 보이지 않는지 확인한다.
- 파일 저장 창의 정상 노트와 전사 실패 노트 상태를 확인한다.
- 긴 한국어·영어 파일명과 좁은 창 크기에서도 레이아웃이 깨지지 않는지 확인한다.

## 12. 지역화 문구

새 문구는 영어와 한국어를 함께 제공한다. 최소 신규 또는 변경 대상은 다음과 같다.

- 재전사 가능한 모델이 필요합니다
- 녹음 파일은 안전하게 보관되어 있습니다.
- 모델 설정에서 로컬 또는 API 전사 모델을 설치하거나 선택하세요. Apple Live는 저장된 녹음의 재전사를 지원하지 않습니다.
- 현재 재전사할 수 있는 모델이 없습니다.
- 전사를 다시 시도할 수 있습니다
- 재전사 버튼을 눌러 다시 시도하세요.
- 복사할 전사 텍스트가 없습니다.
- 파일로 저장
- 저장할 항목과 형식을 선택하세요.
- 전사된 텍스트가 없습니다.
- 저장된 녹음 파일을 찾을 수 없습니다.
- 일반 텍스트 (.txt)
- Markdown (.md)
- 파일 이름
- 저장 위치
- 2개 파일을 저장했습니다.
- 녹음 파일을 저장했습니다.
- 전사 텍스트를 저장했습니다.
- Obsidian으로 내보내기

최종 영어 문구는 구현 시 기존 localization catalog의 어조와 용어를 맞춘다.
