# v0.1.27 릴리스 준비 설계

## 목적

병합된 PR #235의 Note Browser 및 Models 설정 일관성·컨텍스트 처리 개선을 기존 `v0.1.26` 이후의 새 셀프 사인 릴리스로 배포한다.

## 범위

- `version.mk`의 버전을 `0.1.27`, 빌드 번호를 `30`, 태그를 `v0.1.27`로 갱신한다.
- `CHANGELOG.md`에 `0.1.27` 섹션을 추가해 PR #235의 사용자 영향 변경을 기록한다.
- 기존 릴리스 절차를 유지한다. Self-signed Release GitHub Actions는 `v0.1.27`을 입력받아 Quill 인증서로 Universal 앱과 DMG를 서명하고, Sparkle `appcast.xml`, Git 태그, GitHub Release를 생성한다.

## 검증

1. `make -s print-app-version`, `make -s print-build-number`, `make -s print-build-tag`가 각각 `0.1.27`, `30`, `v0.1.27`을 출력해야 한다.
2. `make test`가 통과해야 한다.
3. 준비 커밋을 `main`에 푸시한 뒤 Self-signed Release를 해당 `main` 커밋에서 실행한다.
4. 워크플로가 성공하고 `v0.1.27` 태그, GitHub Release, `Quill.dmg`, `appcast.xml` 자산이 생성되어야 한다.

## 제외 범위

- 릴리스 워크플로, 코드 서명 설정, Sparkle 서명 키의 변경은 포함하지 않는다.
- 이미 배포된 `v0.1.26` 태그나 Release를 수정하거나 재생성하지 않는다.
